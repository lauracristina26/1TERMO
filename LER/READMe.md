# Engenharia de Requisitos: Da Descoberta ao Planejamento Agil 🚀

Este guia aborda as técnicas e metodologias essenciais para identificar o que deve ser construído em um projeto de software.

---

## 1. O Que São Requisitos?
Requisitos são as necessidades e condições que um sistema deve atender para satisfazer os usuários e stakeholders.

### Requisitos Funcionais (RF)
Descrevem **o que** o sistema faz (as funcionalidades).
- *Exemplo:* "O sistema deve permitir o cadastro de novos usuários."

### Requisitos Não Funcionais (RNF)
Descrevem **como** o sistema deve funcionar (desempenho, segurança, usabilidade).
- *Exemplo:* "O sistema deve carregar as páginas em menos de 2 segundos."

---

## 2. Técnicas de Elicitação (Descoberta)

### Brainstorming
Reunião criativa para gerar o máximo de ideias sem julgamentos prévios. Ideal para fases iniciais de descoberta de funcionalidades.

### Entrevista
Conversa direta com stakeholders e usuários para entender suas dores, rotinas e expectativas. Pode ser estruturada ou aberta.

---

## 3. Design Thinking e Design Sprint

### Design Thinking
Abordagem centrada no ser humano para resolver problemas complexos. Foca em:
1. **Empatia** (Entender o usuário).
2. **Definição** (Clarear o problema).
3. **Ideação** (Gerar soluções).
4. **Prototipação**.
5. **Teste**.

### Design Sprint (Google)
Um processo de 5 dias para validar ideias e resolver grandes desafios através de design, prototipagem e teste com usuários. É a "agilização" do Design Thinking.

---

## 4. Prototipagem
Criação de modelos do sistema para validar os requisitos antes do desenvolvimento real.

- **Baixa Fidelidade:** Desenhos em papel (wireframes). Rápido e barato.
- **Alta Fidelidade:** Protótipos interativos (Figma, Adobe XD). Simulam a experiência real do usuário.

---

## 5. Metodologias Ágeis
No contexto de requisitos, a agilidade foca em documentação leve e mudanças constantes conforme o feedback.

- **User Stories:** Forma comum de documentar requisitos (Ex: "Como [usuário], eu quero [ação] para que [benefício]").
- **Backlog:** Lista priorizada de tudo que precisa ser feito.
- **Entrega Contínua:** Requisitos são descobertos e refinados a cada ciclo (Sprint).

---

## Boas Práticas no Levantamento
1. **Envolva o usuário final** desde o início.
2. **Priorize os requisitos** (MoSCoW: Must have, Should have, Could have, Won't have).
3. **Mantenha a rastreabilidade**: Entenda de onde cada requisito surgiu.
